== 데이터 이동 시간 초과 주석

백업 및 복원 중에 작업별 데이터 이동 시간 초과(초)를 설정하려면 어노테이션 `protect.trident.netapp.io/data-mover-timeout-sec`을 사용하십시오. 이 어노테이션은 데이터 이동기가 작업 진행을 기다리는 시간을 지정하며, 이 시간이 지나면 시간 초과됩니다. 이는 전송 중단(예: 네트워크 문제로 볼륨 데이터 복원이 진행되지 않는 경우)을 감지하고 처리하는 데 유용합니다.

아래와 같이 YAML 파일에서 이 주석을 사용할 수 있습니다.

[source, yaml]
----
---
apiVersion: protect.trident.netapp.io/v1
kind: BackupRestore
metadata:
  name: bkp-restore-kopia
  namespace: trident-protect
  annotations: # Optional annotations to configure data movement timeout
    protect.trident.netapp.io/data-mover-timeout-sec: "2"
spec:
  appVaultRef: appvault-name
  appArchivePath: my-backup-path
  namespaceMapping: [{"source": "my-source-namespace", "destination": "my-destination-namespace"}]
  storageClassMapping:
    destination: "${destinationStorageClass}"
    source: "${sourceStorageClass}"
----
이 예시에서, 복원 작업 중 2초 이내에 진행 상황이 없으면 해당 주석은  `BackupRestore` 작업을 실패로 처리합니다.


NOTE: 주석 값이 0보다 작거나 같으면 작업이 즉시 실패합니다.
